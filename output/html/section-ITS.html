Section 4.1 Introduction to Sets

Subsection 4.1.1 Sets

Definition 4.1.1.

Roughly, a set is a collection of distinct objects, called elements (or members).

  • Sets are typically represented by capital letters, like \(A, B, S\text{,}\) etc.

  • The objects in sets can be pretty much anything, i.e. they need not be numbers.

Sets can be defined by listing the elements within set braces, \(\{ \text{ and } \}\text{,}\) separated by commas. For example, the set containing the integers 1, 2, and 4 would be written as,

\begin{equation*} \set{1, 2, 4} \end{equation*}

This is called the \textbf{roster method}, because it involes simply listing the elements of the set.

If an object \(x\) is an element of the set \(A\text{,}\) we write \(x \in A\text{.}\) Otherwise, we write \(x \notin A\) For each object \(x\) and set \(A\text{,}\) either \(x \in A\) or \(x \notin A\text{,}\) but not both.

Example 4.1.2.

Note that \(1 \in \set{1,3}\) (the object 1 is in the set containing 1 and 3), but \(\set{1} \not\in \set{1,3}\) (the set containing 1 is not an element of the set containing 1 and 3). However, \(\set{1} \in \set{\set{1},3}\) (the set containing 1 is an element of the set containing 3 and the set containing 1). In general, an object is not the same as a set containing that object. Also, note that sets are indeed themselves objects, so they can be included among the elements of a set.

Definition 4.1.3.

Two sets are equal, \(A = B\text{,}\) if and only if they contain exactly the same elements (up to ordering). Otherwise, they are not equal, \(A \neq B\text{.}\) In other words, two sets are not equal if at least one element is in one of the sets and is not in the other set.

For example, \(\set{1,2,3} = \set{2,3,1} = \set{3,2,1}\text{,}\) but \(\set{1,2} \neq \set{1,3}\text{.}\)

Aside on definitions.

Definitions are very important in mathematics to facilitate clear communication of ideas. However, it is an unfortunate fact that it is impossible to define every concept. Recall that a set is defined as a “collection of distinct objects”. This naturally leads to the question of how a collection is defined. After all, “set” and “collection” seem to be almost synonyms, so defining one in terms of the other seems unhelpful. Theoretically, we could define “collection”, say as an group or aggregate of things. This isn't much more helpful, since now we would have to define a group or aggregate. This could continue indefinitely, until we run out of suitable words. In this way, a concept like a set is simply defined intuitively.

Subsection 4.1.2 Well-Defined Sets

Sets must be well-defined, in that it must be clear and unambiguous which elements are in the set and which are not. In other words, given any object, it must be clear whether that object is in the set or not.

Example 4.1.4.

The set of all young adults is not well-defined, because it depends on the definition of “young adult”. The set of all people aged 20 or older is well-defined.

Example 4.1.5.

The set of the top 5 airports in the country is not well-defined, because the quality of an airport is subjective and up to interpretation.

Subsection 4.1.3 Duplicate Elements

Note that a set is a collection of distinct objects, so sets with duplicate objects are considered equal to the set without the duplicates. For example,

\begin{equation*} \set{1, 2} = \set{1, 2, 1} = \set{1, 2, 1, 1, 1, 1, 1} \end{equation*}

All of these sets are equal, but of course by convention, sets are written without duplicate elements.

Subsection 4.1.4 Set-Builder Notation

A set can also be defined based on all its elements sharing a common property. This can be done using set-builder notation, which defines a set as,

\begin{equation*} A = \set{x: x \text{ has property } P} \end{equation*}

which means, “\(A\) is the set of all objects \(x\) such that \(x\) has property \(P\)”, or more concisely, “\(A\) is the set of all objects \(x\) with property \(P\)”. Here, the colon \(:\) means “such that”. Sometimes, we use a vertical bar \(|\) instead. Alternatively, it can be written as,

\begin{equation*} A = \set{x \in X: x \text{ has property } P} \end{equation*}

which means, “\(A\) is the set of all objects in \(X\) that satisfy property \(P\)”. Here, the variable \(x\) is a \textit{dummy variable}, in that replacing \(x\) with any other variable or symbol does not change the set. For example,

\begin{equation*} \set{x \in X: x \text{ has property } P} = \set{y \in X: y \text{ has property } P} \end{equation*}

Subsection 4.1.5 The Empty Set

Definition 4.1.6.

The empty set (or null set), denoted by \(\emptyset\) or \(\set{}\text{,}\) is the unique set with no elements.

Subsection 4.1.6 Cardinality

Definition 4.1.7.

Let \(A\) be a set. The cardinality (or size or cardinal number) of a set \(A\text{,}\) denoted by \(n(A)\) or \(\abs{A}\text{,}\) is the number of elements in \(A\text{.}\)

  • The empty set has cardinality zero, \(\abs{\emptyset} = 0\text{,}\) and it is the unique set with this property.
